# Alignment Proposal Template

Use this exact structure before applying changes.

```md
## Alignment proposal for <wiki root>

### Mode

- <full alignment | guidance-only>

### Current shape

- Root: <path>
- Existing schema: <path or "none">
- Existing hub notes: <paths or "none">
- Link styles in use: <wikilinks | markdown links | mixed | plain references>
- Naming patterns: <summary>
- Highest-risk issues: <summary>
- Relevant guidance files: <paths or "none">
- Guidance risks: <summary>

### Proposed alignment

**Create or update:**
  - <path> — <reason>

  In `guidance-only` mode: `none` unless the updated file is itself an `AGENTS.md` or `CLAUDE.md` file.

**Rename or move:**
  - <old path> -> <new path> — <reason>

  In `guidance-only` mode: `none`.

**Link normalization:**
  - <file or scope> — <what will change>

  In `guidance-only` mode: `none`.

**Backlink repairs:**
  - <file or relationship> — <what will change>

  In `guidance-only` mode: `none`.

**Guidance files to update:**
  - <path> — <reason>

**Guidance sections to shorten and point into the wiki:**
  - <path> — <section> -> <wiki page>

**Suspected stale guidance to verify or correct:**
  - <path> — <claim>

**Mirror files left thin or untouched:**
  - <path> — <reason>

**Ambiguities I will not auto-resolve:**
  - <issue or "none">

**Out of scope for this pass:**
  - <issue or "none">

### Safety notes

- <risk or "none">

### Expected result

- <1-3 bullets summarizing the aligned wiki shape>

In `guidance-only` mode, summarize the guidance-file outcome instead of wiki-structure changes.
```
